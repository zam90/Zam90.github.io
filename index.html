【LeetCode笔记】1.两数之和

发表于 2021-12-03 | 分类于 LeetCode笔记
字数统计: 331 | 阅读时长 ≈ 1

[TOC]

问题描述

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出和为目标值 target 的那 两个整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。

你可以按任意顺序返回答案。

【难度】：简单

【语言】：Java

测试用例

输入：nums = [2,7,11,15], target = 9
输出：[0,1]
解释：因为 nums[0] + nums[1] == 9 ，返回 [0, 1] 。
输入：nums = [3,2,4], target = 6
输出：[1,2]
输入：nums = [3,3], target = 6
输出：[0,1]

解法一

本题最易想出的方法就是利用双循环，设置循环变量 i 和 j 分别表示数组里的两个数，计算 i + j 的和与 target 值进行比较，相同则返回输出，不同则继续循环寻找合适的 i 和 j ，直到 i 和 j 都遍历过整个数组。

class Solution {
    public int[] twoSum(int[] nums, int target) {
    for(int i=0;i<nums.length;i++){
        for(int j=0;j<nums.length;j++){
            if((nums[i]+nums[j]==target)&&(i!=j)){
                int[] result={0,0};
                result[0]=i;
                result[1]=j;
                return result;
                }
            }
        }
    return null;
    }
}

n人围圈报数问题

发表于 2021-10-20 | 分类于 学习笔记
字数统计: 794 | 阅读时长 ≈ 3

[TOC]

代码语言

C语言

问题描述

n个人围成一圈，从1-n顺序编号。从第1个人开始循环报数，报到3的人退出圈子(1 2 3 1 2 3…)，问最后留下的人编号是多少？

问题分析

n个人围成一圈报数，最后剩下1个人，所以退出的人数为n-1。

探究一下3人围成一圈报数顺序：第1个人报1，第2个人报2，第3个人报3（3号退出圈子），第1个人报1，第2个人报2，第1个人报3（1号退出圈子）。所以3个人循环报数，最后剩下2号。在这个过程中，需要有一个变量循环访问圈子里的人（第i个人报数），还需要一个变量记录当前报的数（报到了j），同时需要一个一维数组记录第i个人是否在圈子里。

int n = 0;    //人数n
int i = 0;    //循环访问圈子里的人
int j = 0;    //当前报数
int quit = 0;    //退出圈子的人数
int a[999];        //记录第i个人是否在圈子里：a[i]=1表示在圈子中，a[i]=0表示已经退出了圈子

初始化以上变量，将a[n]初值都赋为1（起始的时候n个人都在圈子里），从0-n循环遍历i，直到quit=n-1（退出圈子的人数等于n-1，即只剩1个人）。在循环过程中，每次循环i和j都加1（第i个人报第j个数）；当i循环到第n个人时，将i赋值为0，重新循环遍历；当j=3时，将a[i]置0（该人报到3，退出圈子）。

while (1) {
    //quit=n-1，只剩一个人，退出循环
    if (a[i] == 1 && quit == n - 1) {
        printf("最后1个人编号是%d", i + 1);
        break;
    }
    //报到的不是3，j++
    if (a[i] == 1 && j != 3) {
        j++;
    }
    //报到的是3，该人退出队伍（a[i] = 0），重置j，退出人数quit+1
    if (a[i] == 1 && j == 3) {
        a[i] = 0;
        j = 0;
        quit++;
    }
    i++;
    if (i == n) {
        i = 0;
    }
}

TSF与微服务网关

TSF与API网关结合，API网关就可以封装云应用的业务，将企业系统中的数据、业务逻辑、功能模块安全可靠的开放出来，用来实现系统的集成或与其他系统进行业务链接

微服务网关概述（API网关）

微服务网关是用于实现微服务对外出口的服务，用于协助开发者完成微服务对外提供调用及微服务相互之间调用

微服务网关本身也是一个微服务

API网关的功能

  • 路由转发：通过接收外界的请求，将请求转发到后端相应的微服务
  • 起到过滤器的作用：在API网关中可以完成一系列横向功能（限流、鉴权、监控等）

API网关的优势

  • 减少客户端与微服务之间的调用次数，提高效率
  • 便于监控，可在网关中监控数据，可以做统一的切面任务处理
  • 便于认证，只需要在网关中认证即可，无需每个微服务都进行认证
  • 降低客户端与服务端的耦合度

客户端对API网关发送一次请求，API网关进行认证后就可以调用相应的一系列微服务，无需客户端多次向多个微服务发送请求，降低客户端和服务端耦合度

通过API网关，在不同命名空间下的微服务可以实现相互调用

TSF平台管理

TSF配置管理

配置管理：解决分布式环境下多台服务实例的配置统一管理问题

TSF配置功能的特点

  • 支持配置分布式化管理
  • 支持配置异构系统管理
  • 支持在控制台编写YAML格式配置文件进行配置

TSF分布式配置功能仅支持SpringCloud类型的应用

异构系统：异构系统整合了多个信息化系统的数据库，各数据库在加入异构数据库系统之前本身就已经存在，拥有自己的DBMS

配置管理的分类

  • 应用配置（针对于应用的配置）
  • 全局配置（针对所有微服务配置）
  • 本地配置（针对本地开发时应用的配置）

应用配置和全局配置属于TSF平台的配置（也可称为远程配置、分布式配置）

本地配置是应用在代码工程中创建的配置，如BootStrap.yml

配置优先级：应用配置>全局配置>本地配置。针对于某个应用服务的配置，先寻找针对该应用的配置，没找到按照默认的全局配置进行配置，若也没有全局配置，则按照本地开发时的配置文件进行配置

应用配置

应用配置的功能：

  • 创建配置项：一个配置项可以管理多个版本的配置
  • 生成新版本：基于历史版本生成新版本
  • 发布配置：支持发布配置到部署组
  • 回滚：回滚到上一个版本的配置

全局配置

用于动态更新应用代码中的配置，全局配置可以保证配置内容在某个集群或者命名空间中全局生效

全局配置的功能：

  • 管理配置：创建配置、生成新版本配置、删除配置
  • 配置可以发布到命名空间下的所有应用

传统应用的开发方式

软件开发模式：瀑布开发模型（标准化、阶段清晰、环环相扣）+软件架构：单体架构（开发速度快、成本低）

瀑布开发模型：软件定义（问题定义可行性分析、需求分析）、软件开发（设计、编码、测试）、软件维护（运营维护）

单体架构：最简单的软件架构，功能点+UI界面+后台服务放入同一个包进行开发，然后统一部署

业务驱动下的IT架构变革

  • 资源利用率差

PHP概述

什么是PHP

PHP：超文本预处理器（Hypertext Preprocessor），是一种服务器端、跨平台、HTML嵌入式的脚本语言

PHP是B/S（浏览器/服务器结构）体系结构，属于三层结构。

三层架构就是为了符合“高内聚，低耦合”思想，把各个功能模块划分为表示层（UI）、业务逻辑层（BLL）和数据访问层（DAL）三层架构。

PHP优势

  • 安全性高：开源软件
  • 跨平台特性
  • 支持广泛的数据库
  • 易学性
  • 执行速度快：占用系统资源少
  • 免费：LAMP全部开源免费
  • 模板化：程序逻辑与用户界面分离
  • 支持面向对象与面向过程

PHP应用领域

  • 中小型网站的开发
  • 大型网站的业务逻辑结果展示
  • Web应用系统
  • 硬件管控软件的GUI
  • 电子商务
  • 多媒体系统
  • 企业级应用

适用平台

前言

由于众所周知的原因，Google无法在国内访问。所以国内手机厂商并没有在国产手机上提供Google服务框架，这也导致了安装Google应用变得十分困难，有些游戏在国产手机上因没有Google服务而闪退。

本文仅介绍如何在手机上安装Google服务，不提供虚拟专用网络（VPN）相关内容

安装过程

本文提供极简安装方法，适用于新手小白。

首先下载Google服务安装器（提取码：5oea），然后在手机上安装即可。

安装完成后根据提示，一键安装Google服务。注意安装过程中会多次启用应用安装程序，请注意点击“安装”按钮

所有服务安装完成后，连接国际互联网，然后就可以访问Google Play商店了
