Spring概述

Spring框架概述

  • Spring是轻量级的开源的JavaEE框架
  • Spring可以解决企业应用开发的复杂性
  • Spring有两个核心部门IOC和Aop

IOC：控制反转，把创建对象的过程交给Spring进行管理

Aop：面向切面，不修改源代码进行功能增强

Spring特点

  • 方便解耦，简化开发
  • Aop编程支持
  • 方便程序测试
  • 方便和其他框架进行整合
  • 方便进行事务操作
  • 降低API开发难度

Spring入门实例

导入入门实例需要的基本包

创建User类，添加一个add方法

package com.ZJY.Spring5;

public class User {
    public void add(){
        System.out.println("正在添加......");
    }
}

创建UserBean.xml，并将该xml所在包标记为资源包

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

    <!--创建User对象-->
    <bean id="user" class="com.ZJY.Spring5.User"></bean>
</beans>

创建Test测试类

package com.ZJY.Spring5.Test;

import com.ZJY.Spring5.User;
import org.springframework.context.ApplicationContext;
import org.springframework.context.support.ClassPathXmlApplicationContext;

public class TestUser {
    public void testadd(){
        //加载Spring配置文件
        ApplicationContext context = new ClassPathXmlApplicationContext("UserBean.xml");
        //获取配置创建的对象  "user"是xml中id的值
        User user = context.getBean("user", User.class);
        //输出
        System.out.println(user);
        user.add();
    }
}

IOC控制反转

IOC概念

IOC控制反转，把对象创建和对象之间的调用过程，交给Spring进行管理

使用IOC目的是降低耦合度

入门实例就是IOC的实现

通过控制反转，对象在被创建的时候，由一个调控系统内所有对象的外界实体（xml配置的Spring容器）将其所依赖的对象的引用传递给它。也可以说，依赖被注入到对象中。通过控制反转，业务端从主动创建对象（new对象）变为被动接收对象，其不必关注对象从何而来，如何创建。
