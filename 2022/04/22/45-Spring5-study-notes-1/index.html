Spring概述

Spring框架概述

  • Spring是轻量级的开源的JavaEE框架
  • Spring可以解决企业应用开发的复杂性
  • Spring有两个核心部门IOC和Aop

IOC：控制反转，把创建对象的过程交给Spring进行管理

Aop：面向切面，不修改源代码进行功能增强

Spring特点

  • 方便解耦，简化开发
  • Aop编程支持
  • 方便程序测试
  • 方便和其他框架进行整合
  • 方便进行事务操作
  • 降低API开发难度

Spring入门实例

导入入门实例需要的基本包

创建User类，添加一个add方法

package com.ZJY.Spring5;

public class User {
    public void add(){
        System.out.println("正在添加......");
    }
}

创建UserBean.xml，并将该xml所在包标记为资源包

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

    <!--创建User对象-->
    <bean id="user" class="com.ZJY.Spring5.User"></bean>
</beans>

创建Test测试类

package com.ZJY.Spring5.Test;

import com.ZJY.Spring5.User;
import org.springframework.context.ApplicationContext;
import org.springframework.context.support.ClassPathXmlApplicationContext;

public class TestUser {
    public void testadd(){
        //加载Spring配置文件
        ApplicationContext context = new ClassPathXmlApplicationContext("UserBean.xml");
        //获取配置创建的对象  "user"是xml中id的值
        User user = context.getBean("user", User.class);
        //输出
        System.out.println(user);
        user.add();
    }
}

IOC控制反转

IOC概念

  • IOC控制反转，把对象创建和对象之间的调用过程，交给Spring进行管理
  • 使用IOC目的是降低耦合度
  • 入门实例就是IOC的实现

通过控制反转，对象在被创建的时候，由一个调控系统内所有对象的外界实体（xml配置的Spring容器）将其所依赖的对象的引用传递给它。也可以说，依赖被注入到对象中。通过控制反转，业务端从主动创建对象（new对象）变为被动接收对象，其不必关注对象从何而来，如何创建。

IOC底层原理

IOC原理：xml解析+工厂模式+反射

IOC（接口）

IOC思想基于IOC容器完成，IOC容器底层就是对象工厂

Spring提供IOC容器实现两种方式（两个接口）

（1）BeanFactory：IOC容器基本实现，是Spring内部的使用接口，一般不提供给开发人员使用。加载配置文件时不会创建对象，在获取对象（或使用对象）时才会创建对象。

（2）ApplicationContext：BeanFactory的子接口，提供更强大的功能，一般由开发人员使用。加载配置文件时就会把配置文件对象进行创建。

IOC操作Bean管理

Bean管理指以下两个操作

  • Spring创建对象
  • Spring注入属性

Bean管理操作有两种操作方式

  • 基于xml配置文件方式实现
  • 基于注解方式实现

基于XML方式

基于XML方式创建对象

<!--创建User对象-->
<bean id="user" class="com.ZJY.Spring5.User"></bean>

在Spring配置文件中，使用Bean标签，标签里添加对应属性，就可以实现对象创建

在Bean标签有很多属性，常用属性

  • id属性：对象唯一标识
  • class属性：对象类全路径（包类属性）

name属性：早期属性，与id属性类型，可添加特殊符号

创建对象时，默认执行无参构造方法，若没有无参构造方法，会报错

基于XML方式注入属性

DI：依赖注入（注入属性），DI是IOC一种具体实现

set方法注入

需先创建对象，再注入属性

1.创建user类，定义属性和set方法

package com.ZJY.Spring5;

public class User {
    private int user_id;
    private String user_name;

    public User() {
    }

    //set方法注入
    public void setUser_id(int user_id) {
        this.user_id = user_id;
    }

    public void setUser_name(String user_name) {
        this.user_name = user_name;
    }

    public void show(){
        System.out.println(user_id+","+user_name);
    }
}

2.在xml文件（Spring配置文件）配置对象创建，配置属性注入

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

    <!--2.创建对象，set方法注入属性-->
    <bean id="user" class="com.ZJY.Spring5.User">
        <!--使用property进行属性注入
            name：类里面的属性名称
            value：向属性注入的值   -->
        <property name="user_id" value="123456"/>
        <property name="user_name" value="用户名"/>
    </bean>

</beans>

3.写测试类，进行测试

package com.ZJY.Spring5.Test;

import com.ZJY.Spring5.User;
import org.springframework.context.ApplicationContext;
import org.springframework.context.support.ClassPathXmlApplicationContext;

        User user = context.getBean("user", User.class);
        //输出
        System.out.println(user);
        user.show();
    }
}

有参构造注入

1.创建类、定义属性、创建属性对应的有参构造方法

package com.ZJY.Spring5;

public class User {
    private int user_id;
    private String user_name;

    //有参构造注入
    public User(int user_id, String user_name) {
        this.user_id = user_id;
        this.user_name = user_name;
    }

    public void show(){
        System.out.println(user_id+","+user_name);
    }
}

2.在Spring配置文件中进行配置

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

    <!--3.使用有参构造创建对象，同时进行属性注入-->
    <bean id="user" class="com.ZJY.Spring5.User">
        <!--使用constructor-arg传入构造参数
            name：类里面的属性名称
            value：向属性注入的值   -->
        <constructor-arg name="user_id" value="12345678"/>
        <constructor-arg name="user_name" value="用户名2"/>
    </bean>

</beans>

p名称空间注入

使用不多，用于简化xml配置方式

1.在Spring配置文件中添加p名称空间

<?xml version="1.0" encoding="UTF-8"?>
<!--xml相关约束-->
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:p="http://www.springframework.org/schema/p"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

</beans>

2.在bean标签中，进行属性注入

<?xml version="1.0" encoding="UTF-8"?>
<!--xml相关约束-->
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:p="http://www.springframework.org/schema/p"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

    <!--4.p命名空间注入-->
    <bean id="user" class="com.ZJY.Spring5.User" p:user_id="1234" p:user_name="某用户"/>

</beans>
